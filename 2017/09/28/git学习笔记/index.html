git学习笔记

发表于 2017-09-28 | 更新于 2018-01-13

用作记录，方便查阅

创建git仓库

command description
$ git init 在当前目录初始化仓库

提交/上传文件

command description
$ git add fileName 将工作区文件添加到暂存区
$ git commit -m “some comment” 将暂存区文件添加到版本库

删除文件

command description
$ git rm fileName 删掉暂存区的文件

查看信息

command description
$ git status 显示仓库当前状态
$ git diff fileName 显示工作区和版本库的差异
$ git log [–pretty=oneline] [–graph] [–abbrev-commit] 显示版本库内的commit历史记录 [以单行的形式显示][显示分支图]
$ git reflog 显示版本变更的历史命令
$ git branch 查看当前所在的分支
$ git stash list 查看被储存的工作区列表
$ git remote -v 查看远程库的[详细]信息
$ git tag 查看所有标签
$ git show 查看某一个标签的信息

版本设置

command description
$ git reset –hard HEAD^ HEAD为当前版本，每一个”^”代表上一个版本。HEAD~n代表上n个版本
$ git reset –hard commitId 将当前版本设置到版本号为“commitId”的版本

撤销修改

command description
$ git checkout – fileName 使工作区文件撤销到(暂存区或版本库)的状态
$ git reset HEAD fileName 使暂存区的文件撤销到版本库的状态

远程仓库

command description
$ ssh-keygen -t rsa -C “email@example.com” 获取ssh key
$ git remote add origin git@serverName:path/repoName.git 关联远程库
$ git remote rm origin 删除origin远程库
$ git push -u origin master 第一次推送master分支的所有内容
$ git push origin master 推送最新修改
$ git clone git@serverName:path/repoName.git 克隆远程仓库到本地
$ git push origin master 把本地的master分支推送到远程origin分支上

创建/合并/删除分支

command description
$ git checkout -b dev origin/dev 创建一个dev分支，并关联远程的orgin/dev分支
$ git branch –set-upstream dev origin/dev 设置本地dev于远程仓库origin/dev关联
$ git pull 将本地文件与远程仓库同步
$ git checkout -b dev 创建并切换到分支dev
$ git branch dev 创建分支dev
$ git checkout dev 切换到分支dev
$ git merge dev 把分支dev合并到当前分支
$ git branch -d dev 删除已被合并的分支dev
$ git branch -D dev 强制删除未合并的分支dev
$ git merge –no-ff -m “some comments” dev 以创建一个commit的方式合并dev分支

工作区临时储存/恢复

command description
$ git stash 储存当前工作区状态，并清空
$ git stash apply [stash@{n}] 恢复上一个被储存的工作区 [第n个储存的工作区]
$ git stash drop 删除stash list
$ git stash pop 恢复工作区，并删除stash list

标签创建/删除

command description
$ git tag v1.0 [commit Id] 为当前最新的提交创建名为v1.0的标签 [对给定提交id创建]
$ git tag -a v0.1 -m “some comments” commitId -a指定名字，-m指定说明文字
$ git tag -d v0.1 删除名为v0.1的标签
$ git push origin 将tagName标签推送到远程
$ git push origin –tags 将所有未推送的标签推送到远程
$ git push origin :refs/tags/v0.9 删除远程名为v0.9的标签

git配置

command description
$ git config –global color.ui true 显示颜色
