1 创建git仓库

$ git init

在当前目录初始化仓库

至此文件夹分为三个部分：工作区、暂存区和版本库。

2 提交/上传文件

$ git add fileName

将工作区文件添加到暂存区 

$ git commit -m "some comment"

将暂存区文件添加到版本库

3 删除文件

$ git rm fileName

删掉暂存区的文件

4 查看信息

$ git status

显示仓库当前状态 

$ git diff fileName

显示工作区和版本库的差异 

$ git log [--pretty=oneline] [--graph] [--abbrev-commit]

显示版本库内的commit历史记录 [以单行的形式显示][显示分支图] 

$ git reflog

显示版本变更的历史命令 

$ git branch

查看当前所在的分支 

$ git stash list

查看被储存的工作区列表 

$ git remote -v

查看远程库的[详细]信息 

$ git tag

查看所有标签 

$ git show <tagName>

查看某一个标签的信息

5 版本设置

$ git reset --hard HEAD^

HEAD为当前版本，每一个"^"代表上一个版本。HEAD~n代表上n个版本 

$ git reset --hard commitId

将当前版本设置到版本号为“commitId”的版本

6 撤销修改

$ git checkout -- fileName

使工作区文件撤销到(暂存区||版本库)的状态 

$ git reset HEAD fileName

时暂存区的文件撤销到版本库的状态

7 远程仓库

$ ssh-keygen -t rsa -C "email@example.com"

获取ssh key 

$ git remote add origin git@serverName:path/repoName.git

关联远程库 

$ git remote rm origin

删除origin远程库 

$ git push -u origin master

第一次推送master分支的所有内容 

$ git push origin master

推送最新修改 

$ git clone git@serverName:path/repoName.git

克隆远程仓库到本地 

$ git push origin master

把本地的master分支推送到远程origin分支上 

$ git checkout -b dev origin/dev

创建一个dev分支，并关联远程的orgin/dev分支 

$ git branch --set-upstream dev origin/dev

设置本地dev于远程仓库origin/dev关联 

$ git pull

将本地文件与远程仓库同步

8 创建/合并/删除分支

$ git checkout -b dev

创建并切换到分支dev 

$ git branch dev

创建分支dev 

$ git checkout dev

切换到分支dev 

$ git merge dev

把分支dev合并到当前分支 

$ git branch -d dev

删除已被合并的分支dev 

$ git branch -D dev

强制删除未合并的分支dev 

$ git merge --no-ff -m "some comments" dev

以创建一个commit的方式合并dev分支

9 工作区临时储存/恢复

$ git stash

储存当前工作区状态，并清空 

$ git stash apply [stash@{n}]

恢复上一个被储存的工作区 [第n个储存的工作区] 

$ git stash drop

删除stash list 

$ git stash pop

恢复工作区，并删除stash list

10 标签创建/删除

$ git tag v1.0 [commit Id]

为当前最新的提交创建名为v1.0的标签 [对给定提交id创建] 

$ git tag -a v0.1 -m "some comments" commitId

-a指定名字，-m指定说明文字 

$ git tag -d v0.1

删除名为v0.1的标签 

$ git push origin <tagName>

将tagName标签推送到远程 

$ git push origin --tags

将所有未推送的标签推送到远程 

$ git push origin :refs/tags/v0.9

删除远程名为v0.9的标签

11 git配置

$ git config --global color.ui true

显示颜色 

$ git config --global alias.lg "log --color --graph
--pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset 
%s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

good…

当前配置文件位置：yourrepo/.git/config

全局配置文件位置：~/.gitconfig
